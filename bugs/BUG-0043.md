# BUG-0043: Review mode loses completed morning session status

**Status:** FIXED
**Severity:** P1
**Reported:** 2026-04-13
**Fixed in:** pending commit
**Reporter:** Jiong

## Description
After tapping "Review" on a completed morning session, exiting the review causes the Practice tab to show the morning session as incomplete/available instead of completed.

## Steps to Reproduce
1. Complete morning session
2. Practice tab shows "Morning Session Complete ✓" with Review button
3. Tap "Review"
4. Exit review (X → Pause & Exit, or complete it)
5. Expected: Practice tab still shows "Morning Session Complete ✓"
6. Actual: Practice tab shows morning as available/incomplete

## Root Cause
Two issues found:
1. `pause()` in `SessionFlowViewModel` was writing to SQLite even in review mode, creating a new paused session record that overrode the completed one
2. `navigateToReview` state wasn't being cleared on return, potentially causing re-navigation issues

## Fix
1. Added `guard !isReviewMode else { return }` to `pause()` method — review mode never writes session state
2. Added `navigateToReview = nil` cleanup in `onAppear` of PracticeTabView
3. All review mode database writes already guarded: `loadWords()`, `recordAnswer()`, `completeSession()`, `autoSaveProgress()`, now `pause()` too

## Files Changed
- `ViewModels/SessionFlowViewModel.swift` — guard in pause()
- `Views/Practice/PracticeTabView.swift` — clear navigateToReview on appear
